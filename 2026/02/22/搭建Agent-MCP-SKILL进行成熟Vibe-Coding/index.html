搭建Agent+MCP+SKILL进行成熟Vibe Coding

2.1k 8 分钟
# AI # SKILL

Vibe Coding自提出来，就有各大互联网公司、模型厂商以及开发者相应，提出许多基础优化编码。从模型的训练Function Calling能力和上下文长度提升,到Prompt技术，RAG增强，再到MCP统一协议获取信息，再到最新的SKILL。从Copilot,Cursor,Claude Code,OpenCode,Trae,国内外各大公司和开源产品都纷纷自己实现了相关协议或者提出了相关功能。这里就我比较常用的几个编码IDE进行实践，使得能够更加成熟方便地Vibe Coding.

省流版:个人推荐vscode+copilot或者Zed+Opencode

如果嫌麻烦直接上Trae. 此外还有Windsurf,Qoder以及google antigravity等等,这里使用不多就不做过多介绍

Vscode+Copilot

事实上Vscode可以搭配claude code,gemini cli以及qwen code等等工具。但这里主要还是使用Copilot,因为其与vscode搭配最丝滑，也提供了最新的功能体验。这也是我目前使用最多的AI编程工具了。

相比于Cursor和Claude code,vscode的商业属性没有那么浓

MCP

个人觉得vscode安装并使用MCP比Cursor方便多了。MCP Registry

在 VS Code 中使用 MCP 服务器 - VSCode · AI 代码编辑器

在扩展中使用@mcp搜索即可下载,在聊天中而可以通过#显示提醒，或者直接进行询问，Agent会进行查找

安装MCP可以通过npm,pip等工具进行.

SKILL

skill是可复用的经验和流程，一般都是在文件系统中创建md文件，包括例子，流程和资源等。安装就是将对应md文件放在某个目录中,或者使用npx skills add进行添加

  1. SkillsMP（全球最大）
  • 地址：https://skillsmp.com
  • 规模：14 万 + Skills，覆盖代码 / 办公 / 设计 / 自动化
  • 兼容：Trae、Cursor、Claude、VS Code
  • 用法：搜 → 下载 ZIP → 拖入 Trae → 自动解压到.trae/skills
  1. Skills.sh（Vercel 官方商店）
  • 地址：https://skills.sh
  • 规模：1.25 万 +，一键安装、按下载量 / 趋势排序
  • 兼容：Trae、Claude、Cursor、Windsurf
  • 用法：复制命令 / 链接 → 粘贴到 Trae → 自动安装
  1. Smithery.ai
  • 地址：https://smithery.ai/skills
  • 规模：1.5 万 +，显示激活次数 / GitHub Stars
  • 特点：社区驱动、质量高、可直接导出 ZIP
  • 兼容：Trae、Claude、Cursor

📦 官方 / 开源仓库（标准模板、可复制）

  1. Anthropic 官方 Skills（标准源头）
  • GitHub：https://github.com/anthropics/skills
  • 内容：16+ 官方示例（代码审查、文档处理、前端设计）
  • 格式：标准SKILL.md，Trae 直接识别
  • 用法：克隆 / 下载文件夹 → 放入.trae/skills
  1. OpenAI Skills（官方模板）
  • GitHub：https://github.com/openai/skills
  • 特点：通用 Agent Skills 标准，适合跨平台复用
  1. AgentSkills 官方索引
  • 地址：https://agentskills.io
  • 作用：Skills 标准文档 + 社区索引，跳转 GitHub 仓库

🧩 精选社区合集（高质量、可直接用）

  1. Awesome Claude Skills
  • GitHub：https://github.com/ComposioHQ/awesome-claude-skills
  • 规模：300+ 高质量 Skills，持续更新
  • 用法：复制链接 / 下载 → 导入 Trae
  1. AI Templates（aitmpl）
  • 地址：https://aitmpl.com/skills
  • 特点：254+ 精选模板，企业级配置、Stack Builder
  1. SkillStore.io（中文友好）
  • 地址：https://skillstore.io/zh-hans/skills
  • 特点：中文分类、国内访问快、Trae 友好

目前能用 Skills 的主流客户端：

排序 工具名 是否免费使用Skills 推荐人群 技能存放默认路径 备注
1 Claude Code 是（官方） 所有人 ~/.claude/skills 标准制定者，生态最全
2 Cursor 是 写代码最常用 ~/.cursor/skills 几乎无缝兼容Claude Skills
3 Trae / OpenCode 是 追求性价比 看工具设置 国内用户较多
4 VS Code + 插件 部分支持 已经深度用vscode 插件设置里配置 正在快速跟进
5 扣子/其他国内平台 部分支持 喜欢网页版 平台自带技能市场 有的要会员

自定义代理

VS Code 中的自定义代理 - VSCode · AI 代码编辑器

Claude Code

claude code可以作为扩展集成到vscode上，也可以使用终端cli工具，免费版每天50条消息左右。Claude Code 入门教程 | 菜鸟教程

但其优点是可以更方便搭配其他API站点和密钥进行使用，例如Claude Code+GLM或者Deepseek,Qwen等。

国产大模型接入 Claude Code 教程：以 Doubao-Seed-Code 为例 - 阮一峰的网络日志

配置后可以在vscode开启终端使用,本质就是更改api endpoint和api key. 目前可以搭配豆包,GLM,KIMI,DeepSeek等适配Claude Code API输出的格式。

Cursor

Cursor目前免费额度是基础代码补全（Tab 补全）：2000 次 / 月

高级模型慢速请求（GPT‑4/Claude 等）：50 次 / 月

Cursor‑small 轻量模型：200 次 / 月 Pro版20刀一个月，如果没有经济压力还是推荐直接升级为Pro.

API Key配置

不过可以自己配置API key,包括OpenAI,Anthropic(也就是Claude系列),Google(Gemini系列)以及Azure等(Azure也提供了学生一年免费服务，我之前也用过，很不错)。 所以完全可以考虑使用第三方中转API key进行配置，中转站有很多，为了避免广告嫌疑，我这里就不提了。

注意这里Anthropic的API地址不允许重写，所以如果要用,要么使用支持Anthropic官方API的中转站，要么使用OpenAI处的重写BaseURL进行配置，可以自定义添加模型(但目前免费版只能使用Auto,因此免费版要么只选择自定义model,否则其调用哪个模型是不确定的),配置好后即可使用

Cursor中将Rules,Skills,Agents,MCP Servers等都统称为插件Plugins | Cursor Docs,官方有插件市场可供安装，也可以本地配置。Cursor - Marketplace

Zed+OpenCode

OpenCode是开源的免费Agent,可以搭配一些模型提供商，而Zed主要是使用起来比较轻量，可以添加自定义LLM provider以及MCP。但目前说实话，各种功能和使用体验还是比不上Vscode。好处是集成了许多可用的Agent。OpenCode 入门教程 | 菜鸟教程

国内翘楚Trae

国内的AI IDE/CLI主要是字节，阿里和腾讯，分别的代表性产品就是Trae,Qwen Code和CodeBuddy. 这里主要以Trae作说明，其更新比较快，使用体验目前也是最好。目前国内版还没有推出付费，但海外版有了SOLO付费模式。优点是不需要自己配置太多东西，开箱即用。

SOLO 模式以 AI 为核心主导，全程自动规划与执行从需求理解、代码生成、测试到成果预览的全开发流程。在 SOLO 模式下， 我们只需通过自然语言描述、语音沟通或上传本地文件等灵活方式提交需求，AI 会快速自主拆解任务、高效推进执行，让开发过程变得极度简化、全程智能化。

SOLO模式下还推出了SOLO Coder和SOLO Builder
